# Kaspi Web — Super App (демо-прототип)

Реализация по техническому заданию «Веб-версия Kaspi Super App» (v1.0) — адаптивный
(mobile-first) веб-сайт, имитирующий ключевые функции приложения Kaspi.kz.

> ⚠️ **Это учебный демонстрационный прототип.** Он **не связан** с АО «Kaspi Bank»,
> не имеет реальных интеграций (банк, eGov, SMS, биометрия) и работает исключительно
> на **вымышленных данных**. Не вводите настоящие персональные, банковские или
> государственные данные. Всё сохраняется только в `localStorage` вашего браузера.

## Стек

- **React 18 + Vite** (frontend). Бэкенд не требуется — данные моковые, состояние в `localStorage`.
- `qrcode` — генерация QR-кодов для документов и реквизитов.
- Чистый CSS, mobile-first, без UI-библиотек.

## Запуск

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # продакшн-сборка в dist/
```

Демо-вход: любой номер телефона → код из SMS **`0000`**.

## Установка на телефон (PWA)

Приложение — устанавливаемое PWA: `manifest.webmanifest`, иконки (192/512/maskable +
apple-touch-icon) и `service worker` (офлайн-старт оболочки). После установки открывается
в полноэкранном режиме с красной иконкой.

- **Android/Chrome**: меню → «Установить приложение» (нужен HTTPS).
- **iOS/Safari**: «Поделиться» → «На экран Домой».
- ⚠️ Service worker и предложение установки требуют **HTTPS** (или `localhost`).
  По локальному IP на `http://` установка полноценно не сработает — деплойте прод-сборку
  (`npm run build` → `dist/`) на HTTPS-хостинг.

## Деплой на GitHub Pages

В репозитории есть workflow `.github/workflows/deploy.yml`: при push в `main` он собирает
проект и публикует на Pages. `base` подставляется автоматически как `/<repo>/`, поэтому
проект работает по адресу `https://<user>.github.io/<repo>/`.

1. Создайте репозиторий на GitHub и запушьте ветку `main`.
2. **Settings → Pages → Build and deployment → Source: GitHub Actions**.
3. Дождитесь workflow «Deploy to GitHub Pages» — в конце будет ссылка на сайт.

> ⚠️ Pages из **приватного** репозитория доступен только на платных планах
> (GitHub Pro/Team/Enterprise). На бесплатном плане репозиторий должен быть публичным.
> Для корневого домена или страницы `<user>.github.io` оставьте `base` = `/`.

## Навигация (как в приложении Kaspi)

- **Главная** — лаунчер: промо-баннеры рассрочки, сетка сервисов 4×2
  (Kaspi QR · Мой Банк · Платежи · Переводы · Магазин · Travel · Госуслуги · Объявления),
  мерчант-промо и продуктовые карточки.
- **Нижняя навигация**: Главная · Kaspi QR · Сообщения · Сервисы.
- **Госуслуги** открываются заставкой «eGov + Kaspi», далее документы и пошлины.
- **Документ** — вкладки «Документ» (загрузка файла) и «Реквизиты» (форма ФИО/ИИН/…),
  сохранение чёрной кнопкой «Сохранить →».

## Структура

```
src/
  App.jsx            навигация (лаунчер + 4 таба), состояние, persistence
  data.js            демо-данные (сетка, промо, карты, документы, услуги)
  utils.js           форматирование, localStorage, генерация QR
  components/
    Icon.jsx         набор SVG-иконок (вместо эмодзи)
    Login.jsx        демо-авторизация (SMS-код 0000)
    Home.jsx         главная-лаунчер: сетка сервисов, промо, продукты
    Bank.jsx         «Мой Банк»: карты, бюджет, транзакции, управление картой
    Gov.jsx          «Госуслуги»: eGov-заставка, документы (вкладки), пошлины
    Requisites.jsx   реквизиты для переводов: форма, QR, копирование
    Sheet.jsx        нижняя модалка (bottom sheet)
```

## Примечание по безопасности

Настоящая интеграция с банковскими и государственными системами требует лицензий,
защищённых серверных API и сертифицированной аутентификации. Такой функционал в
демо намеренно не реализован — экраны входа, оплаты и документов лишь имитируют
интерфейс на фейковых данных.
